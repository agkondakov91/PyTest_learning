### -v / --verbose — подробный вывод

С флагом `-v` — pytest показывает не только точки и буквы, а полные имена тестов и статус каждого:

```bash
uv run pytest -v

# tests/test_calculator.py::test_add PASSED
# tests/test_calculator.py::test_divide PASSED
# tests/test_calculator.py::test_divide_by_zero PASSED
```

### -k — запуск по имени

Чтобы `pytest` запустит только тесты, в имени которых есть определённое слово, необходимо использовать флаг `-k`:

```bash
uv run pytest -v -k "add"

# tests/test_calculator.py::test_add PASSED
# tests/test_calculator.py::test_add_negative PASSED
# tests/test_calculator.py::test_add_zero PASSED
```

### -x / --exitfirst — остановиться при первом падении

По умолчанию pytest прогоняет все тесты до конца, даже если часть из них уже упала. Флаг `-x` **останавливает выполнение сразу после первого упавшего теста**.

```bash
uv run pytest -x

# Остановиться после первых двух падений
uv run pytest --maxfail=2
```

### -s — показывать print() в тестах

По умолчанию `pytest` перехватывает весь вывод в консоль. 

Если вы написали **print() **внутри теста для отладки — вы его не увидите.

Флаг `-s` отключает перехват.

```bash
uv run pytest -s
```

### Запуск конкретного файла или теста

Мы можем запускать конкретные файлы и без флагов – по названию файла с тестами:

```bash
# Запустить только один файл
uv run pytest tests/test_calculator.py

# Запустить один конкретный тест
uv run pytest tests/test_calculator.py::test_add

# Запустить несколько тестов из файла
uv run pytest tests/test_calculator.py::test_add tests/test_calculator.py::test_divide
```
